# Evidence Rubric — strong / adequate / weak / missing

For audit grading. Each clause gets one of four verdicts:

## STRONG (100 pts)
- Document or evidence exists at the expected path
- All required IDs / sections present
- Linked to upstream + downstream artifacts (bidirectional traceability)
- An auditor would accept without follow-up questions

## ADEQUATE (75 pts)
- Most required evidence present but with 1 documented gap
- Document exists but missing 1 sub-section
- Evidence is partial (e.g., 80% test coverage where >=95% expected)
- Auditor will likely ask clarifying questions

## WEAK (40 pts)
- Minimal evidence present
- Document is a stub
- Required evidence exists in code but not in the expected document
- Auditor will flag as a finding

## MISSING (0 pts)
- No evidence found
- Required document does not exist
- Immediate audit failure on this clause
